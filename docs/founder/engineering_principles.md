# Engineering Philosophy

Genesis follows a philosophy of deliberate engineering.

The purpose of engineering is not merely to produce software.

The purpose of engineering is to produce maintainable, understandable, extensible, and trustworthy systems.

---

## No Placeholders

Placeholder implementations create false confidence.

Every implementation must represent genuine functionality.

---

## No Technical Debt By Neglect

Tradeoffs may be accepted.

Undocumented debt may not.

---

## Documentation Is Part Of The Product

A feature that only exists in code does not fully exist.

Every significant capability must be documented.

---

## Testing Is A Requirement

Testing is not an enhancement.

Testing is evidence.

Assertions without evidence are not accepted.

---

## Reversibility Matters

The ability to recover is more valuable than the ability to move quickly.

---

## Discoverability Matters

Every component should be understandable.

Every file should have a purpose.

Every dependency should be justified.

---

## Knowledge Is Infrastructure

Code is one form of knowledge.

Documentation, ADRs, diagrams, governance, tests, and architecture are equally important forms of infrastructure.

Engineering must treat them accordingly.
