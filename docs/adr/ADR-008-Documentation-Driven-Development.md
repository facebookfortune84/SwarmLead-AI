# ADR-008

Title: Documentation Driven Development

Status: Accepted

Date: 2026-07-23

---

## Context

Software frequently accumulates undocumented decisions.

This produces operational risk, onboarding friction, and knowledge loss.

Genesis seeks to preserve understanding alongside implementation.

---

## Decision

Genesis shall adopt Documentation Driven Development.

Significant changes should be documented before and during implementation.

Documentation artifacts include:

- ADRs
- Architecture Documents
- Specifications
- Process Documentation
- Governance Artifacts
- Knowledge Graph Updates

Documentation is considered a core deliverable.

Code completion alone is insufficient.

Features are not considered complete until:

- implementation exists
- tests exist
- documentation exists
- architectural impacts are recorded

---

## Consequences

Benefits:

- Better maintainability
- Reduced onboarding friction
- Increased organizational memory
- Improved architectural consistency

Costs:

- Additional documentation effort

The tradeoff is accepted.
