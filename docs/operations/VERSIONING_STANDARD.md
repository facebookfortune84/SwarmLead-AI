# Versioning Standard

## Strategy

Semantic Versioning

MAJOR.MINOR.PATCH

Example:

2.5.12

---

## Major

Breaking changes.

Example:

1.x.x → 2.x.x

---

## Minor

New features.

Example:

1.4.x → 1.5.x

---

## Patch

Bug fixes.

Example:

1.5.3 → 1.5.4

---

## Release Tag Format

vMAJOR.MINOR.PATCH

Example:

v2.0.0

---

## Principle

Version numbers communicate change risk.
